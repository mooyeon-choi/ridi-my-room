# 기획서 (Design Document)

## 1. 프로젝트 개요

### 1.1 서비스 제목
리디 마이룸

### 1.2 장르
- 게이미피케이션 마이 페이지

### 1.3 플랫폼
- Web

### 1.4 타겟 사용자
- 리디북스 사용자
- 독서 기록을 시각적으로 즐기고 싶은 사용자
- 꾸미기/커스터마이징을 좋아하는 사용자
- 레트로/픽셀아트 감성을 선호하는 사용자

### 1.5 핵심 컨셉
트래블러스 레스트 스타일의 픽셀 아트 그래픽을 사용한 내 서재 마이페이지. 게이미피케이션 요소를 활용해서 내 서재를 동물의 숲 내방 꾸미기, 싸이월드 미니홈피처럼 꾸밀 수 있으며, AI 아바타가 사용자를 대신해 방문자와 인터랙션함.

### 1.6 MVP 범위 (간단한 버전)
- 간단한 서재 공간 (책상, 책장, 창문)
- 기본 아바타 움직임
- AI 기반 방문자 응대
- 최소한의 커스터마이징 옵션

## 2. 주요 기능

### 2.1 핵심 사용자 플로우
1. **서재 입장** - 내 픽셀 아트 서재 공간 확인
2. **독서 활동 반영** - 읽은 책이 서재에 자동으로 배치됨
3. **꾸미기** - 가구/소품 배치로 서재 커스터마이징
4. **방문/교류** - 다른 사용자의 서재 방문, 내 서재에 방문자 받기
5. **AI 아바타 상호작용** - 방문 시 AI가 주인을 대신해 응대

### 2.2 주요 기능

#### 2.2.1 서재 꾸미기 시스템
- **기본 가구 (MVP)**:
  - 책상 (1개)
  - 책장 (1-2개)
  - 창문 (1-2개)
- **가구 배치**: 드래그 앤 드롭으로 위치 조정
- **향후 확장**: 조명, 러그, 식물, 액자 등

#### 2.2.2 독서 연동 시스템
- **책 자동 배치**: 리디에서 읽은 책이 서재 책장에 표시됨
- **읽은 책 시각화**: 읽은 페이지/완독 여부에 따라 시각적 변화
- **독서 통계**: 독서량, 선호 장르 등을 서재 분위기에 반영

#### 2.2.3 아바타 시스템
- **내 아바타**: 서재 내에서 NPC처럼 자율적으로 이동
  - Idle 상태: 책 읽기, 창문 바라보기, 책장 정리 등
  - 방문자 있을 때: 인사, 반응 애니메이션
- **아바타 커스터마이징**: 헤어, 의상, 액세서리 (향후 확장)

#### 2.2.4 AI 상호작용 시스템
- **AI 인사**: 방문자가 들어오면 자동 인사
- **AI 채팅 응답**:
  - 방문자가 채팅 메시지를 남기면 AI가 주인 대신 답변
  - 사용자의 독서 취향, 프로필 정보 기반 개인화된 응답
  - 자연스러운 대화 톤 유지
- **응답 설정**:
  - AI 페르소나 설정 (친근함/공손함/유머러스 등)
  - 자동 응답 ON/OFF

#### 2.2.5 방문 시스템
- **다른 서재 방문**: 친구/랜덤 사용자 서재 둘러보기
- **방명록**: 메시지 남기기
- **좋아요/리액션**: 서재 꾸미기에 반응 남기기
- **방문 기록**: 누가 방문했는지 확인

### 2.3 게이미피케이션 요소
- **리워드 시스템**:
  - 책 읽을수록 포인트 획득
  - 포인트로 가구/소품 구매
- **도전과제**:
  - "한 달에 5권 읽기" → 특별 가구 획득
  - "10명에게 방문 받기" → 특별 장식 획득
- **레벨 시스템**:
  - 독서량에 따라 레벨업
  - 레벨별 잠금 해제 콘텐츠

## 3. 아트 스타일

### 3.1 비주얼 방향
- **아트 스타일**: 픽셀 아트 (Traveler's Rest 스타일)
- **해상도**:
  - 타일 크기: 16x16px or 32x32px
  - 캔버스: 세로모드 기준 반응형 웹 (기본 비율 9:16, 최소 360x640px)
- **색상 팔레트**:
  - 따뜻한 톤 (나무, 책의 아늑함 표현)
  - 차분한 중간톤 중심
- **애니메이션**:
  - 아바타: 걷기(4방향), Idle 동작(책 읽기, 스트레칭 등)
  - 환경: 창문 밖 풍경, 촛불 흔들림(옵션)
  - 부드러운 2-4 프레임 애니메이션

### 3.2 UI/UX
- **인터페이스 스타일**:
  - 픽셀 아트와 어울리는 레트로 UI
  - 리디 브랜드 컬러 포인트 활용
- **화면 방향**: 세로모드 (Portrait) 전용
- **메뉴 구조**:
  - 상단 바: 프로필, 알림, 설정
  - 하단 탭 바: 내 서재, 둘러보기, 꾸미기, 독서 통계
- **편집 모드**:
  - 꾸미기 ON/OFF 토글
  - 하단 인터페이스 영역에 가구 선택 패널 표시
  - 상단 배경 영역에서 드래그 앤 드롭 배치

### 3.3 서재 공간 디자인 (MVP)

#### 에셋 경로
- 배경 이미지: `public/assets/backgrounds/`
  - `background_1.png` — 기본 서재 배경

#### 기본 레이아웃 (세로모드 - 상하 분할)
```
┌───────────────┐
│               │
│  배경 영역     │  ← 상단: 서재 배경 (background_1.png)
│  (너비 100%   │     너비에 맞춰 채우고, 비율 유지
│   비율 유지)   │     아바타, 가구, 책장 등 표시
│               │
├───────────────┤
│               │
│  인터페이스    │  ← 하단: 채팅, 방명록, 독서 통계 등
│  영역         │     UI 패널 (탭 전환)
│               │
│               │
└───────────────┘
    [하단 탭 바]     ← 내 서재 / 둘러보기 / 꾸미기 / 통계
```
- **상단 (배경 영역)**: 서재 배경 이미지를 화면 너비에 맞춰 표시. 아바타 활동, 가구 배치, 책장 등 게임 렌더링 영역
- **하단 (인터페이스 영역)**: 채팅/AI 대화, 방명록, 독서 통계 등 상호작용 UI

#### 필수 오브젝트 (MVP)
1. **책상** (1개)
   - 캐릭터가 앉아서 책 읽는 애니메이션
   - 현재 읽고 있는 책 표시
2. **책장** (1-2개)
   - 읽은 책들이 쌓임
   - 클릭하면 책 목록 확인 가능
3. **창문** (1-2개)
   - 배경 분위기 연출
   - 시간대별 밝기 변화 (선택사항)

#### 향후 확장 가능 오브젝트
- 조명 (스탠드, 샹들리에)
- 러그/카펫
- 화분/식물
- 액자/포스터
- 소파/의자
- 사이드 테이블

## 4. 기술 스펙

### 4.1 프론트엔드
- **프레임워크**: React / Next.js (추천)
- **게임 렌더링 엔진**:
  - Phaser.js (픽셀 아트 게임에 최적화)
  - 또는 PixiJS (경량화)
  - 또는 Kaboom.js (간단한 구현)
- **상태 관리**: Zustand / Recoil
- **스타일링**: Tailwind CSS + 커스텀 픽셀 UI

### 4.2 백엔드
- **API**: RESTful API 또는 GraphQL
- **데이터베이스**: PostgreSQL / MongoDB
- **파일 스토리지**: AWS S3 / Cloudflare R2 (스프라이트, 아바타 이미지)

### 4.3 AI 통합
- **AI 채팅 엔진**:
  - OpenAI GPT API
  - 또는 Claude API
  - 또는 자체 LLM 파인튜닝
- **페르소나 관리**: 사용자 독서 데이터 기반 프롬프트 생성
- **응답 캐싱**: 자주 묻는 질문 캐싱으로 비용 절감

## 5. 콘텐츠 설계

### 5.1 아바타 시스템

#### 기본 아바타 (MVP)
- **외형**: 남/녀 기본 1종씩
- **기본 동작**:
  - 걷기 (상하좌우 4방향)
  - Idle (책 읽기, 창문 바라보기)
  - 인사 (방문자에게 손 흔들기)

#### AI 페르소나 타입
| 타입 | 말투 | 특징 | 예시 응답 |
|------|------|------|-----------|
| 친근함 | 반말, 이모지 사용 | 편안하고 다정한 톤 | "어서와! 우리 서재 구경해줘서 고마워 😊" |
| 공손함 | 존댓말, 정중함 | 격식있고 정중한 톤 | "방문해주셔서 감사합니다. 천천히 둘러보세요." |
| 유머러스 | 농담, 재치있는 표현 | 위트있고 재미있는 톤 | "책벌레의 아지트에 온 걸 환영해! ㅋㅋ" |
| 독서광 | 책/문학 관련 표현 | 책과 독서에 대한 열정 | "좋은 책은 마음의 양식이죠. 추천 드릴까요?" |

### 5.2 가구/오브젝트 (MVP)

#### 기본 가구 (무료 제공)
| 이름 | 설명 | 크기(타일) | 상호작용 |
|------|------|-----------|----------|
| 기본 책상 | 심플한 나무 책상 | 2x1 | 아바타가 앉아서 책 읽기 |
| 기본 책장 | 작은 책장 | 1x2 | 읽은 책 자동 배치 |
| 기본 창문 | 작은 사각 창문 | 1x1 | 배경 조명 효과 |

#### 확장 가구 (포인트/도전과제)
| 이름 | 획득 방법 | 가격 | 효과 |
|------|-----------|------|------|
| 대형 책장 | 10권 완독 | 500P | 더 많은 책 진열 가능 |
| 고급 책상 | 레벨 5 달성 | 800P | 아바타 애니메이션 추가 |
| 아치형 창문 | 방문자 20명 | 1000P | 서재 분위기 UP |
| 스탠드 조명 | 한 달 연속 독서 | 600P | 밤 분위기 연출 |
| 북엔드 세트 | 장르별 5권 완독 | 300P | 책장 꾸미기 |
| 독서 의자 | 레벨 3 | 400P | 아바타 앉기 포즈 |

### 5.3 책 시각화

#### 책장 배치 규칙
- 완독한 책: 책등이 보이게 꽂힘
- 읽는 중인 책: 책상 위에 펼쳐진 상태
- 최근 읽은 책: 하이라이트 효과

#### 책 표현 방식
- 장르별 색상 구분 (판타지-보라, SF-파랑, 로맨스-핑크 등)
- 두께는 페이지 수 반영
- 베스트셀러는 골드 테두리

### 5.4 서재 테마 (향후 확장)
1. **기본 서재** (무료)
   - 따뜻한 나무 톤
2. **모던 서재** (1500P)
   - 화이트/그레이 미니멀
3. **클래식 서재** (2000P)
   - 고급스러운 다크 우드
4. **판타지 서재** (2500P)
   - 마법사의 서재 느낌

## 6. 사운드 디자인 (선택사항)

### 6.1 배경 음악 (BGM)
- **메인 BGM**: 잔잔한 로파이/재즈 (집중 방해하지 않는 음악)
- **방문자 도착**: 짧은 알림 징글
- **도전과제 달성**: 성취감 있는 효과음

### 6.2 효과음 (SFX)
- **UI 상호작용**:
  - 버튼 클릭: 부드러운 클릭음
  - 가구 배치: 툭 놓는 소리
  - 메뉴 열기/닫기: 슬라이드 소리
- **아바타 동작**:
  - 걷기: 발걸음 소리 (타일별 다름)
  - 책장 정리: 책 넘기는 소리
- **환경음** (옵션):
  - 시계 초침 소리
  - 창밖 새소리/빗소리

### 6.3 음향 설정
- BGM/SFX 개별 볼륨 조절
- 음소거 옵션
- 집중 모드 (사운드 OFF)

## 9. 메모 및 아이디어

### 차별화 포인트
- **독서 활동과 직접 연동**: 읽은 책이 실제로 서재에 쌓이는 경험
- **AI 아바타**: 내가 없을 때도 방문자를 응대하는 나만의 AI
- **픽셀 아트 감성**: 레트로하면서도 따뜻한 비주얼
- **독서 커뮤니티**: 서재 방문을 통한 자연스러운 독서 교류

### 향후 확장 아이디어
- **소셜 기능 강화**
  - 친구 추가/팔로우 시스템
  - 책 추천 교환
  - 같이 읽기 챌린지
- **커스터마이징 확대**
  - 아바타 외형 다양화
  - 서재 테마 추가
  - 계절별 이벤트 장식
- **리디 서비스 통합 깊화**
  - 특정 책 구매 시 관련 가구 증정
  - 작가 사인회 참여 → 특별 액자 획득
  - 독서 토론 참여 → 포인트 적립
- **모바일 앱**
  - 푸시 알림 (방문자 알림)
  - 간단한 채팅 확인/응답

### 기술적 고려사항
- **성능 최적화**
  - 스프라이트 아틀라스 사용
  - 레이지 로딩
  - 방문자 수 제한 (동시 접속)
- **AI 비용 관리**
  - 응답 캐싱
  - 토큰 사용량 제한
  - 무료/유료 티어 구분
- **보안**
  - 부적절한 채팅 필터링
  - 스팸 방지
  - 개인정보 보호

### 우려사항 & 해결 방안
- **우려**: AI 응답 품질 편차
  - **해결**: 프롬프트 엔지니어링 + 피드백 루프
- **우려**: 초기 콘텐츠 부족 (가구가 적음)
  - **해결**: MVP는 간단하게, 점진적 업데이트
- **우려**: 사용자 이탈 (방문이 뜸함)
  - **해결**: 알림, 도전과제로 재방문 유도

### 성공 지표 (KPI)
- MAU (월간 활성 사용자)
- 평균 서재 방문 횟수
- 가구 구매/배치 비율
- AI 채팅 만족도
- 독서량 증가율
